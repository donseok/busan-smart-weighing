## 1. 시스템 소개

### 1.1 개요

**부산 스마트 계량 시스템**은 부산 제철소의 차량 계량 업무를 자동화하는 통합 솔루션입니다. LPR(차량 번호판 인식) 기술과 AI를 결합하여 차량이 계량대에 진입하면 자동으로 번호판을 인식하고, 배차 정보를 매칭하여 계량을 수행합니다.

본 시스템은 기존 수기 작성 기반의 계량 업무를 전자화하여 업무 효율성을 높이고, 데이터의 정확성과 투명성을 확보하는 것을 목표로 합니다.

### 1.2 주요 특징

| 특징 | 설명 |
|------|------|
| **LPR 자동 계량** | AI 기반 차량 번호판 자동 인식(정확도 95% 이상)으로 무인 계량 수행 |
| **모바일 OTP 백업** | LPR 인식 실패 시 모바일 OTP를 통한 대체 계량 지원 |
| **전자 계량표** | 종이 계량표를 대체하는 전자 계량표 자동 생성 및 공유 |
| **실시간 모니터링** | WebSocket 기반 실시간 장비 상태 및 계량 현황 모니터링 |
| **통계/분석** | 일별, 월별, 품목별, 업체별 계량 통계 및 Excel 다운로드 |

### 1.3 시스템 구성

본 시스템은 다음 두 가지 접근 방식을 제공합니다.

- **웹 관리 시스템**: PC 브라우저를 통해 접속하는 관리 화면 (주로 담당자용)
- **모바일 앱**: 스마트폰에 설치하여 사용하는 앱 (주로 운전자용)

### 1.4 사용자 역할

본 시스템은 세 가지 사용자 역할을 지원합니다. 본 매뉴얼은 **운전자**와 **담당자** 역할을 중심으로 안내합니다.

| 역할 | 설명 | 주요 사용 환경 |
|------|------|----------------|
| **관리자 (ADMIN)** | 시스템 전체 설정 및 사용자 관리 | 웹 |
| **담당자 (MANAGER)** | 배차, 계량, 출문, 기준정보, 통계 관리 | 웹 |
| **운전자 (DRIVER)** | 배차 확인, 계량 수행, 전자 계량표 확인 | 모바일 앱 |

### 1.5 계량 업무 흐름

전체 계량 업무는 다음과 같은 단계로 진행됩니다.

```
배차 등록 → 차량 입차 → 1차 계량(공차) → 상/하차 작업 → 2차 계량(실차) → 전자 계량표 발행 → 출문
```

**참고**: 품목 유형에 따라 3차 계량이 추가될 수 있으며, 순중량은 |1차 중량 - 2차 중량|으로 자동 계산됩니다.

### 2.2 로그인

#### 2.2.1 웹 로그인 (담당자)

1. 브라우저를 열고 시스템 주소에 접속합니다.
2. **로그인 화면** (`/login`)이 표시됩니다.
3. **아이디** 입력란에 발급받은 사용자 ID를 입력합니다.
4. **비밀번호** 입력란에 비밀번호를 입력합니다.
5. **로그인** 버튼을 클릭합니다.
6. 인증 성공 시 **대시보드** 화면(`/dashboard`)으로 이동합니다.

> **주의사항**
> - 비밀번호를 5회 연속 잘못 입력하면 계정이 **30분간 잠금** 처리됩니다.
> - 계정이 잠긴 경우 30분 후 자동으로 해제되며, 즉시 해제가 필요하면 관리자에게 문의하십시오.
> - 로그인 후 **30분간** 조작이 없으면 자동 로그아웃됩니다(액세스 토큰 만료). 7일 이내 재접속 시 자동 갱신됩니다.

#### 2.2.2 모바일 앱 로그인 (운전자)

1. 스마트폰에서 **부산 스마트 계량** 앱을 실행합니다.
2. **아이디**와 **비밀번호**를 입력합니다.
3. **로그인** 버튼을 탭합니다.
4. 인증 성공 시 메인 화면으로 이동합니다.

> **참고**: 모바일 앱은 자동 로그인 기능을 지원합니다. 최초 로그인 후 7일간 자동 로그인이 유지됩니다(리프레시 토큰 기반).

#### 2.2.3 모바일 OTP 로그인

LPR 자동 인식이 실패한 경우, 현장의 LED 전광판에 표시되는 OTP(일회용 비밀번호)를 통해 계량을 수행할 수 있습니다.

1. 계량대 진입 후 LED 전광판에 표시되는 **6자리 OTP 코드**를 확인합니다.
2. 모바일 앱에서 **OTP 계량** 메뉴를 선택합니다.
3. 6자리 OTP 코드를 입력합니다.
4. **확인** 버튼을 탭합니다.

> **주의사항**
> - OTP는 발급 후 **5분** 이내에 입력해야 합니다. 시간 초과 시 새로운 OTP가 발급됩니다.
> - OTP를 **3회 연속** 잘못 입력하면 해당 OTP는 무효화됩니다. 계량 담당자에게 문의하십시오.

### 2.3 로그아웃

#### 웹 로그아웃

1. 화면 우측 상단의 **사용자 아이콘**을 클릭합니다.
2. 드롭다운 메뉴에서 **로그아웃**을 선택합니다.
3. 로그인 화면(`/login`)으로 이동합니다.

#### 모바일 앱 로그아웃

1. 하단 내비게이션에서 **마이페이지** 탭을 선택합니다.
2. **로그아웃** 버튼을 탭합니다.
3. 확인 팝업에서 **확인**을 탭합니다.

## 3. 운전자 기능

운전자는 주로 **모바일 앱**을 통해 시스템을 이용합니다. 배차 조회, 계량 수행, 전자 계량표 확인 및 공유가 주요 기능입니다.

### 3.1 배차 조회

배차 조회 기능을 통해 자신에게 배정된 배차(운송 건)를 확인할 수 있습니다.

#### 배차 조회 방법

1. 모바일 앱 하단 내비게이션에서 **배차** 탭을 선택합니다.
2. 배차 목록이 표시됩니다. 각 항목에는 다음 정보가 포함됩니다.

   | 항목 | 설명 |
   |------|------|
   | 배차 번호 | 고유 배차 식별 번호 |
   | 상태 | 등록(REGISTERED), 진행중(IN_PROGRESS), 완료(COMPLETED), 취소(CANCELLED) |
   | 품목 유형 | 부산물, 폐기물, 부재료, 반출, 일반 |
   | 상차지/하차지 | 출발지 및 도착지 정보 |
   | 배차일자 | 배차가 예정된 날짜 |
   | 차량 번호 | 배정된 차량의 번호판 |

3. 특정 배차를 탭하면 **배차 상세 화면**으로 이동하여 세부 정보를 확인할 수 있습니다.

#### 배차 상태 설명

```
등록(REGISTERED) → 진행중(IN_PROGRESS) → 완료(COMPLETED)
                                        → 취소(CANCELLED)
```

| 상태 | 설명 |
|------|------|
| **등록 (REGISTERED)** | 담당자가 배차를 등록한 초기 상태. 아직 계량이 시작되지 않음 |
| **진행중 (IN_PROGRESS)** | 1차 계량이 시작되어 계량 프로세스가 진행 중인 상태 |
| **완료 (COMPLETED)** | 모든 계량이 완료되고 전자 계량표가 발행된 상태 |
| **취소 (CANCELLED)** | 배차가 취소된 상태. 사유는 상세 화면에서 확인 가능 |

> **참고**: 배차 목록은 기본적으로 최근 배차일자 순으로 정렬됩니다. 상단의 필터를 사용하여 상태별로 조회할 수 있습니다.

### 3.2 LPR 자동 계량

LPR(License Plate Recognition) 자동 계량은 차량이 계량대에 진입하면 AI가 자동으로 번호판을 인식하여 계량을 수행하는 핵심 기능입니다. 운전자가 별도로 조작할 필요 없이 자동으로 진행됩니다.

#### 자동 계량 절차

다음은 차량이 계량대에 진입한 후 자동으로 진행되는 과정입니다.

**단계 1: 차량 진입 감지**
- 차량이 계량대에 진입하면 **LiDAR(라이다) 센서**가 차량의 존재를 자동으로 감지합니다.
- LED 전광판에 "차량 감지 중..." 메시지가 표시됩니다.

**단계 2: 번호판 인식**
- LPR 카메라가 차량 번호판을 촬영합니다.
- AI 엔진이 번호판 이미지를 분석하여 차량 번호를 인식합니다 (정확도 95% 이상).

**단계 3: 배차 자동 매칭**
- 인식된 차량 번호를 기반으로 시스템이 등록된 배차 정보를 자동으로 검색하고 매칭합니다.
- 매칭 성공 시 LED 전광판에 차량 번호와 배차 정보가 표시됩니다.

**단계 4: 중량 측정**
- 계량 지시기(인디케이터)가 RS-232C 통신을 통해 중량 데이터를 읽어옵니다.
- 중량 값이 안정되면 자동으로 측정값이 확정됩니다.

**단계 5: 데이터 저장 및 계량표 발행**
- 측정된 중량이 자동으로 서버에 저장됩니다.
- 해당 계량이 최종 계량(2차 또는 3차)인 경우, **전자 계량표**가 자동 생성됩니다.
- 모바일 앱에 알림이 전송됩니다.

**단계 6: 출차**
- LED 전광판에 "계량 완료" 메시지와 측정 중량이 표시됩니다.
- 차단기가 열리면 안전하게 출차합니다.

#### 운전자 확인 사항

자동 계량이 원활하게 진행되려면 다음 사항을 확인하십시오.

1. 차량 **번호판이 깨끗한 상태**인지 확인합니다 (오염, 가림 시 인식률 저하).
2. 계량대 위에 차량을 **정확하게 정차**합니다.
3. 계량 중에는 차량의 **시동을 끄지 않고 대기**합니다.
4. LED 전광판의 안내 메시지를 확인하며 지시에 따릅니다.
5. "계량 완료" 메시지가 표시된 후에 출차합니다.

> **주의**: 계량 중에 차량에서 하차하거나 차량을 이동하면 정확한 중량 측정이 불가능합니다. 반드시 계량 완료 안내를 확인한 후 출차하십시오.

#### 계량 순서 안내

| 계량 차수 | 설명 | 일반적인 상태 |
|-----------|------|---------------|
| **1차 계량** | 첫 번째 중량 측정 (일반적으로 공차 중량) | 상차 전 또는 하차 전 |
| **2차 계량** | 두 번째 중량 측정 (일반적으로 실차 중량) | 상차 후 또는 하차 후 |
| **3차 계량** | 필요 시 추가 계량 | 특수 품목 또는 재계량 |

> **참고**: **순중량(Net Weight)** = |1차 중량 - 2차 중량| 으로 자동 계산됩니다.

### 3.3 모바일 OTP 계량

LPR 자동 인식이 실패하거나 특수한 상황에서는 모바일 OTP를 사용하여 계량을 수행할 수 있습니다.

#### OTP 계량이 필요한 경우

- 번호판 오염, 손상, 가림 등으로 LPR 인식이 실패한 경우
- 임시 번호판을 사용하는 차량인 경우
- 시스템 일시 오류로 자동 인식이 불가한 경우

#### OTP 계량 절차

**단계 1: OTP 코드 확인**
1. 차량이 계량대에 진입한 후 LPR 인식이 실패하면, LED 전광판에 **6자리 OTP 코드**가 표시됩니다.
2. 표시된 OTP 코드를 메모하거나 기억합니다.

**단계 2: 모바일 앱에서 OTP 입력**
1. 모바일 앱을 실행합니다.
2. 메인 화면 또는 계량 탭에서 **OTP 계량** 버튼을 탭합니다.
3. 6자리 OTP 코드를 입력합니다.
4. **확인** 버튼을 탭합니다.

**단계 3: 본인 확인**
1. 등록된 휴대전화 번호로 본인 확인이 진행됩니다.
2. 확인이 완료되면 다음 단계로 자동 진행됩니다.

**단계 4: 배차 선택**
1. 본인에게 배정된 배차 목록이 표시됩니다.
2. 현재 계량할 **배차 건**을 선택합니다.
3. **계량 시작** 버튼을 탭합니다.

**단계 5: 계량 수행**
1. 시스템이 계량 지시기와 연동하여 중량을 측정합니다.
2. 측정된 중량이 앱 화면에 표시됩니다.
3. 중량 값을 확인하고 **확인** 버튼을 탭합니다.

**단계 6: 계량 완료**
1. 계량 데이터가 서버에 저장됩니다.
2. 최종 계량인 경우 전자 계량표가 자동 생성됩니다.
3. LED 전광판에 "계량 완료" 메시지가 표시됩니다.
4. 안전하게 출차합니다.

> **주의사항**
> - OTP 코드는 발급 후 **5분간**만 유효합니다. 시간 초과 시 LED 전광판의 새 OTP를 확인하십시오.
> - OTP를 **3회 연속** 잘못 입력하면 해당 OTP가 무효화됩니다. 이 경우 계량 담당자(물류관제실)에 문의하여 수동 처리를 요청하십시오.
> - OTP 계량 중 앱이 종료되면 처음부터 다시 시도해야 합니다.

### 3.4 전자 계량표 확인 및 공유

계량이 완료되면 전자 계량표가 자동으로 생성됩니다. 운전자는 모바일 앱에서 계량표를 확인하고 공유할 수 있습니다.

#### 3.4.1 전자 계량표 확인

1. 모바일 앱 하단 내비게이션에서 **계량** 탭을 선택합니다.
2. 완료된 계량 목록이 표시됩니다.
3. 확인하려는 계량 건을 탭합니다.
4. **전자 계량표 보기** 버튼을 탭합니다.
5. 전자 계량표가 화면에 표시됩니다.

전자 계량표에는 다음 정보가 포함됩니다.

| 항목 | 설명 |
|------|------|
| 계량표 번호 | 고유 식별 번호 |
| 배차 번호 | 관련 배차 번호 |
| 차량 번호 | 계량 대상 차량 번호 |
| 운전자 정보 | 운전자명, 소속 업체 |
| 품목 유형 | 부산물, 폐기물, 부재료, 반출, 일반 |
| 1차 중량 | 첫 번째 계량 값 (kg) |
| 2차 중량 | 두 번째 계량 값 (kg) |
| 3차 중량 | 세 번째 계량 값 (해당 시, kg) |
| 순중량 | 자동 계산된 순중량 (kg) |
| 계량 일시 | 각 차수별 계량 일시 |
| 계량 모드 | LPR(자동), 모바일(OTP), 수동(manual) |
| 비고 | 특이사항 기재란 |

#### 3.4.2 전자 계량표 공유

전자 계량표를 거래처 또는 관계자에게 공유할 수 있습니다.

**카카오톡으로 공유하기**

1. 전자 계량표 상세 화면에서 **공유** 버튼을 탭합니다.
2. 공유 방법 선택 화면에서 **카카오톡**을 선택합니다.
3. 카카오톡 앱이 실행되며 공유할 대상(친구 또는 채팅방)을 선택합니다.
4. **전송** 버튼을 탭합니다.

**SMS로 공유하기**

1. 전자 계량표 상세 화면에서 **공유** 버튼을 탭합니다.
2. 공유 방법 선택 화면에서 **SMS**를 선택합니다.
3. 수신자 전화번호를 입력하거나 연락처에서 선택합니다.
4. **전송** 버튼을 탭합니다.

> **참고**: 공유 시 전자 계량표의 핵심 정보(계량표 번호, 차량 번호, 순중량, 계량 일시)와 상세 내용을 확인할 수 있는 링크가 포함됩니다.

### 3.5 즐겨찾기

자주 확인하는 배차나 업체를 즐겨찾기에 등록하여 빠르게 접근할 수 있습니다.

#### 즐겨찾기 등록

1. 배차 목록, 업체 정보 등 원하는 항목의 상세 화면으로 이동합니다.
2. 화면의 **별 아이콘**(☆)을 탭합니다.
3. 아이콘이 채워진 별(★)로 변경되면 즐겨찾기에 등록된 것입니다.
4. 다시 탭하면 즐겨찾기가 해제됩니다.

#### 즐겨찾기 목록 확인

1. 모바일 앱 메인 화면(홈)의 **즐겨찾기** 영역을 확인합니다.
2. 등록된 즐겨찾기가 유형별(배차, 업체, 차량)로 표시됩니다.
3. 항목을 탭하면 해당 상세 화면으로 바로 이동합니다.

> **참고**: 즐겨찾기 목록의 순서를 변경할 수 있습니다. 목록에서 항목을 길게 누르면 드래그하여 순서를 조정할 수 있습니다.

### 3.6 알림 확인

시스템에서 발송하는 알림을 통해 계량 관련 주요 정보를 실시간으로 받을 수 있습니다.

1. 모바일 앱 상단의 **알림 아이콘**(종 모양)을 탭합니다.
2. 알림 목록이 표시됩니다.
3. 읽지 않은 알림은 강조 표시됩니다.
4. 알림을 탭하면 해당 화면(배차 상세, 계량표 등)으로 이동합니다.

주요 알림 유형:

| 알림 유형 | 내용 |
|-----------|------|
| 배차 알림 | 새로운 배차가 배정되었을 때 |
| 계량 완료 | 자동 계량이 완료되었을 때 |
| 계량표 발행 | 전자 계량표가 생성되었을 때 |
| 공지사항 | 새로운 공지사항이 등록되었을 때 |
| 시스템 알림 | 계정 관련 안내, 시스템 점검 공지 등 |

### 3.7 문의하기 (전화 문의)

시스템 사용 중 문제가 발생하거나 도움이 필요한 경우 문의 기능을 이용하여 담당 부서에 직접 전화 연결할 수 있습니다.

1. 모바일 앱 메인 화면 또는 마이페이지에서 **문의하기**를 탭합니다.
2. 문의 대상을 선택합니다.

   | 문의 대상 | 담당 업무 |
   |-----------|-----------|
   | **물류관제실** | 계량 관련 문의, OTP 오류, 장비 이상 등 |
   | **자재창고** | 품목 관련 문의, 상/하차 관련 문의 등 |

3. 선택한 부서의 **전화 연결** 버튼을 탭하면 전화 앱이 실행됩니다.
4. 통화가 종료되면 앱으로 돌아오며, **통화 기록**이 자동으로 저장됩니다.
5. 이전 통화 이력은 문의하기 화면의 **내 통화 기록** 목록에서 확인할 수 있습니다.

> **참고**: 긴급한 상황(장비 고장, 안전 문제 등)에서는 즉시 물류관제실에 전화하십시오.

### 3.8 계량 이력 조회

모바일 앱에서 자신의 과거 계량 및 배차 이력을 확인할 수 있습니다.

1. 모바일 앱 하단 내비게이션에서 **홈** 탭을 선택합니다.
2. **이력 조회** 메뉴를 탭합니다.
3. 기간별 계량 완료 건수, 배차 이력이 표시됩니다.
4. 특정 건을 탭하면 해당 계량 또는 배차의 상세 정보를 확인할 수 있습니다.

## 4. 담당자 기능

담당자는 **웹 관리 시스템**을 통해 배차, 계량, 출문, 기준정보, 통계, 모니터링 등 전반적인 계량 업무를 관리합니다.

### 4.1 대시보드

대시보드(`/dashboard`)는 로그인 후 가장 먼저 표시되는 화면으로, 계량 업무의 전반적인 현황을 한눈에 파악할 수 있습니다.

#### 대시보드 구성 요소

| 구성 요소 | 설명 |
|-----------|------|
| **오늘의 계량 현황** | 금일 총 계량 건수, 완료 건수, 진행 중 건수 표시 |
| **월간 계량 현황** | 당월 누적 계량 건수 및 중량 합계 |
| **실시간 차트** | 시간대별 계량 추이를 실시간 그래프로 표시 (ECharts 기반) |
| **업체별 통계** | 거래 업체별 계량 실적 요약 (건수, 중량) |
| **진행 중 계량** | 현재 계량대에서 진행 중인 계량 목록 (실시간 업데이트) |
| **공지사항** | 최근 등록된 공지사항 요약 (최대 5건, 고정 공지 우선 표시) |
| **장비 상태 요약** | 계량기, LPR 카메라, LiDAR 등 주요 장비 상태 |

#### 대시보드 활용 방법

1. 좌측 메뉴에서 **대시보드**를 클릭하거나, 로그인 직후 자동으로 표시됩니다.
2. 각 현황 카드를 클릭하면 해당 상세 화면으로 이동합니다.
   - "오늘의 계량 현황" 클릭 시 계량 관리(`/weighing`) 화면으로 이동
   - "진행 중 계량" 항목 클릭 시 해당 계량 상세 정보 표시
   - "공지사항" 클릭 시 공지사항(`/notice`) 화면으로 이동
3. 실시간 차트는 **WebSocket** 기반으로 자동 업데이트되므로 새로고침이 필요하지 않습니다.

> **참고**: 대시보드의 데이터는 실시간으로 갱신됩니다. 화면에 이상이 있으면 브라우저를 새로고침(F5)하십시오.

### 4.2 배차 관리

배차 관리(`/dispatch`)는 차량 운송 건을 등록하고 관리하는 기능입니다.

#### 4.2.1 배차 등록

1. 좌측 메뉴에서 **배차 관리**를 클릭합니다.
2. 배차 목록 화면 우측 상단의 **신규 등록** 버튼을 클릭합니다.
3. 배차 등록 양식에 다음 정보를 입력합니다.

   | 입력 항목 | 필수 여부 | 설명 |
   |-----------|-----------|------|
   | 배차일자 | 필수 | 배차 예정일 (달력 선택) |
   | 품목 유형 | 필수 | 부산물, 폐기물, 부재료, 반출, 일반 중 선택 |
   | 업체 | 필수 | 기준정보에 등록된 업체 선택 |
   | 차량 번호 | 필수 | 기준정보에 등록된 차량 선택 |
   | 운전자 | 필수 | 해당 차량에 배정된 운전자 선택 |
   | 상차지 | 필수 | 출발지(적재 장소) |
   | 하차지 | 필수 | 도착지(하역 장소) |
   | 비고 | 선택 | 특이사항 기재 |

4. 입력 내용을 확인한 후 **저장** 버튼을 클릭합니다.
5. "배차가 등록되었습니다." 확인 메시지가 표시됩니다.
6. 등록된 배차는 **등록(REGISTERED)** 상태로 목록에 추가됩니다.

> **주의**: 동일 차량에 대해 같은 날짜에 진행 중인 배차가 있는 경우, 중복 경고 메시지가 표시될 수 있습니다. 확인 후 등록을 진행하십시오.

#### 4.2.2 배차 조회 및 검색

1. 좌측 메뉴에서 **배차 관리**를 클릭합니다.
2. 배차 목록이 표 형태로 표시됩니다.
3. 상단의 검색 조건을 활용하여 원하는 배차를 찾을 수 있습니다.

   | 검색 조건 | 설명 |
   |-----------|------|
   | 기간 | 배차일자 기준 시작일 ~ 종료일 |
   | 상태 | 등록, 진행중, 완료, 취소 (복수 선택 가능) |
   | 품목 유형 | 부산물, 폐기물, 부재료, 반출, 일반 |
   | 업체 | 업체명으로 검색 |
   | 차량 번호 | 차량 번호로 검색 |
   | 운전자 | 운전자명으로 검색 |

4. 조건을 입력한 후 **검색** 버튼을 클릭합니다.
5. **초기화** 버튼을 클릭하면 모든 검색 조건이 초기화됩니다.

#### 4.2.3 배차 수정

1. 배차 목록에서 수정할 배차 건을 클릭하여 상세 화면으로 이동합니다.
2. **수정** 버튼을 클릭합니다.
3. 수정이 필요한 항목의 값을 변경합니다.
4. **저장** 버튼을 클릭합니다.

> **주의**: **진행중(IN_PROGRESS)** 또는 **완료(COMPLETED)** 상태의 배차는 수정할 수 없습니다. 수정이 필요한 경우 관리자에게 문의하십시오.

#### 4.2.4 배차 삭제

1. 배차 목록에서 삭제할 배차 건을 클릭하여 상세 화면으로 이동합니다.
2. **삭제** 버튼을 클릭합니다.
3. "정말 삭제하시겠습니까?" 확인 팝업이 표시됩니다.
4. **확인**을 클릭하면 삭제가 완료됩니다.

> **주의**: **등록(REGISTERED)** 상태의 배차만 삭제할 수 있습니다. 삭제된 데이터는 복구할 수 없으므로 신중하게 처리하십시오.

#### 4.2.5 배차 상태 변경

배차 상태는 계량 진행에 따라 자동으로 변경됩니다. 필요 시 수동으로 상태를 변경할 수 있습니다.

| 현재 상태 | 변경 가능 상태 | 설명 |
|-----------|----------------|------|
| 등록(REGISTERED) | 취소(CANCELLED) | 배차 취소 처리 |
| 진행중(IN_PROGRESS) | 취소(CANCELLED) | 진행 중 배차 취소 (사유 입력 필수) |

1. 배차 상세 화면에서 **상태 변경** 버튼을 클릭합니다.
2. 변경할 상태를 선택합니다.
3. 취소 처리 시 **취소 사유**를 입력합니다.
4. **확인** 버튼을 클릭합니다.

> **참고**: 등록 상태에서 진행중 상태로의 변경은 1차 계량이 시작될 때 시스템이 자동으로 처리합니다. 완료 상태로의 변경 역시 최종 계량 완료 시 자동 처리됩니다.

### 4.3 계량 관리

계량 관리(`/weighing`)는 모든 계량 기록을 조회하고 관리하는 기능입니다.

#### 4.3.1 계량 기록 조회

1. 좌측 메뉴에서 **계량 관리**를 클릭합니다.
2. 계량 기록 목록이 표시됩니다.
3. 상단 검색 조건을 활용하여 조회합니다.

   | 검색 조건 | 설명 |
   |-----------|------|
   | 기간 | 계량 일시 기준 시작일 ~ 종료일 |
   | 계량 모드 | LPR(자동), 모바일(OTP), 수동(manual) |
   | 품목 유형 | 부산물, 폐기물, 부재료, 반출, 일반 |
   | 차량 번호 | 차량 번호로 검색 |
   | 업체 | 업체명으로 검색 |
   | 상태 | 진행중, 완료 |

4. **검색** 버튼을 클릭합니다.

#### 4.3.2 계량 상세 정보

계량 목록에서 특정 건을 클릭하면 상세 정보를 확인할 수 있습니다.

| 정보 구분 | 상세 항목 |
|-----------|-----------|
| **기본 정보** | 계량 번호, 배차 번호, 차량 번호, 운전자, 업체, 품목 유형 |
| **1차 계량** | 1차 중량(kg), 계량 일시, 계량 모드(LPR/모바일/수동), 계량기 정보 |
| **2차 계량** | 2차 중량(kg), 계량 일시, 계량 모드, 계량기 정보 |
| **3차 계량** | 3차 중량(kg), 계량 일시, 계량 모드, 계량기 정보 (해당 시) |
| **결과** | 순중량(|1차 - 2차|), 계량 완료 여부 |
| **LPR 정보** | 인식된 번호판 이미지, 인식 신뢰도(%), AI 모델 버전 |
| **비고** | 특이사항, 수정 이력 |

#### 4.3.3 수동 계량

자동(LPR) 및 모바일(OTP) 계량이 모두 불가한 경우, 담당자가 직접 수동으로 계량 데이터를 입력할 수 있습니다.

1. 계량 관리 화면에서 **수동 계량** 버튼을 클릭합니다.
2. 다음 정보를 입력합니다.

   | 입력 항목 | 필수 여부 | 설명 |
   |-----------|-----------|------|
   | 배차 선택 | 필수 | 진행 중인 배차 목록에서 선택 |
   | 계량 차수 | 필수 | 1차, 2차, 3차 중 선택 |
   | 중량 | 필수 | 계량 지시기에 표시된 중량 직접 입력 (kg) |
   | 계량기 | 필수 | 사용 계량기 선택 |
   | 사유 | 필수 | 수동 계량 사유 입력 |

3. 입력 내용을 확인한 후 **저장** 버튼을 클릭합니다.

> **주의**: 수동 계량은 자동/모바일 계량이 불가한 경우에만 사용하십시오. 수동 계량 기록에는 "수동" 표시와 함께 사유가 기록됩니다.

#### 4.3.4 재계량

이미 측정된 계량에 대해 재계량이 필요한 경우 다음과 같이 처리합니다.

1. 계량 상세 화면에서 **재계량** 버튼을 클릭합니다.
2. 재계량 사유를 입력합니다.
3. **확인** 버튼을 클릭합니다.
4. 해당 차수의 계량이 초기화되고, 차량이 다시 계량대에 진입하면 새로운 계량이 수행됩니다.

> **참고**: 재계량 이력은 시스템에 기록되며, 이전 계량 데이터도 이력으로 보존됩니다.

### 4.4 전자 계량표 관리

전자 계량표 관리(`/slip`)에서는 발행된 모든 전자 계량표를 조회하고 공유 이력을 관리할 수 있습니다.

#### 4.4.1 전자 계량표 조회

1. 좌측 메뉴에서 **전자 계량표**를 클릭합니다.
2. 발행된 전자 계량표 목록이 표시됩니다.
3. 검색 조건(기간, 차량 번호, 업체, 품목 유형)을 활용하여 조회합니다.
4. 목록에서 특정 계량표를 클릭하면 상세 내용이 표시됩니다.

#### 4.4.2 공유 이력 확인

1. 전자 계량표 상세 화면에서 **공유 이력** 탭을 클릭합니다.
2. 해당 계량표의 공유 이력이 표시됩니다.

   | 이력 항목 | 설명 |
   |-----------|------|
   | 공유 일시 | 공유된 날짜와 시간 |
   | 공유 방법 | 카카오톡, SMS 등 |
   | 수신자 | 공유 대상 정보 |
   | 공유자 | 공유를 실행한 사용자 |

### 4.5 출문 관리

출문 관리(`/gate-pass`)는 차량의 제철소 출문(퇴장)을 승인하거나 반려하는 기능입니다.

#### 4.5.1 출문 승인

1. 좌측 메뉴에서 **출문 관리**를 클릭합니다.
2. 승인 대기 중인 출문 요청 목록이 표시됩니다.
3. 승인할 출문 건을 클릭하여 상세 화면으로 이동합니다.
4. 다음 정보를 확인합니다.

   | 확인 항목 | 설명 |
   |-----------|------|
   | 배차 정보 | 배차 번호, 품목 유형, 업체 |
   | 차량 정보 | 차량 번호, 운전자 |
   | 계량 정보 | 계량 완료 여부, 순중량 |
   | 전자 계량표 | 계량표 발행 여부 |

5. 확인이 완료되면 **승인** 버튼을 클릭합니다.
6. "출문이 승인되었습니다." 확인 메시지가 표시됩니다.

#### 4.5.2 출문 반려

1. 출문 상세 화면에서 **반려** 버튼을 클릭합니다.
2. **반려 사유**를 입력합니다 (필수).
3. **확인** 버튼을 클릭합니다.
4. 해당 운전자에게 반려 알림이 전송됩니다.

> **주의**: 출문 승인은 계량이 정상적으로 완료되고 전자 계량표가 발행된 건에 대해서만 가능합니다. 계량 미완료 건에 대한 출문은 반드시 반려 처리하십시오.

### 4.6 기준정보 관리

기준정보 관리는 시스템 운영에 필요한 기본 데이터를 관리하는 기능입니다. 좌측 메뉴의 **기준정보**를 클릭하면 하위 메뉴가 표시됩니다.

#### 4.6.1 업체 관리

업체 관리(`/master/company`)에서는 거래처 정보를 등록하고 관리합니다.

**업체 등록**

1. 좌측 메뉴에서 **기준정보 > 업체**를 클릭합니다.
2. **신규 등록** 버튼을 클릭합니다.
3. 다음 정보를 입력합니다.

   | 입력 항목 | 필수 여부 | 설명 |
   |-----------|-----------|------|
   | 업체코드 | 필수 | 고유 업체 식별 코드 |
   | 업체명 | 필수 | 업체 이름 |
   | 사업자등록번호 | 필수 | 10자리 사업자등록번호 |
   | 대표자명 | 선택 | 대표자 이름 |
   | 전화번호 | 선택 | 업체 연락처 |
   | 주소 | 선택 | 업체 소재지 |
   | 사용 여부 | 필수 | 사용/미사용 선택 |

4. **저장** 버튼을 클릭합니다.

**업체 조회/수정/삭제**

- **조회**: 업체 목록에서 업체명 또는 업체코드로 검색하여 조회합니다.
- **수정**: 업체를 클릭하여 상세 화면 진입 후 **수정** 버튼을 클릭하여 정보를 변경합니다.
- **삭제**: 상세 화면에서 **삭제** 버튼을 클릭합니다. 해당 업체에 연결된 배차/계량 기록이 있으면 삭제할 수 없으며, 대신 **미사용** 처리합니다.

#### 4.6.2 차량 관리

차량 관리(`/master/vehicle`)에서는 계량 대상 차량 정보를 등록하고 관리합니다.

**차량 등록**

1. 좌측 메뉴에서 **기준정보 > 차량**을 클릭합니다.
2. **신규 등록** 버튼을 클릭합니다.
3. 다음 정보를 입력합니다.

   | 입력 항목 | 필수 여부 | 설명 |
   |-----------|-----------|------|
   | 차량 번호 | 필수 | 차량 번호판 번호 (예: 12가3456) |
   | 차량 유형 | 필수 | 덤프트럭, 카고트럭, 탱크로리 등 |
   | 소속 업체 | 필수 | 기준정보에 등록된 업체 선택 |
   | 운전자 | 선택 | 기본 배정 운전자 |
   | 공차 중량 | 선택 | 빈 차량의 기본 중량 (kg) |
   | 사용 여부 | 필수 | 사용/미사용 선택 |

4. **저장** 버튼을 클릭합니다.

> **참고**: 차량 번호는 LPR 자동 인식 시 배차 매칭에 사용되는 핵심 정보입니다. 정확하게 입력하십시오.

**차량 조회/수정/삭제**

- **조회**: 차량 목록에서 차량 번호, 업체명, 차량 유형으로 검색합니다.
- **수정**: 차량을 클릭하여 상세 화면 진입 후 **수정** 버튼을 클릭합니다.
- **삭제**: 상세 화면에서 **삭제** 버튼을 클릭합니다. 연결된 데이터가 있으면 삭제 불가하며, **미사용** 처리합니다.

#### 4.6.3 저울(계량기) 관리

저울 관리(`/master/scale`)에서는 현장의 계량기 장비 정보를 등록하고 관리합니다.

**저울 등록**

1. 좌측 메뉴에서 **기준정보 > 저울**을 클릭합니다.
2. **신규 등록** 버튼을 클릭합니다.
3. 다음 정보를 입력합니다.

   | 입력 항목 | 필수 여부 | 설명 |
   |-----------|-----------|------|
   | 저울코드 | 필수 | 고유 저울 식별 코드 |
   | 저울명 | 필수 | 저울 이름 (예: 1번 계량기, 부두 계량기) |
   | 설치 위치 | 필수 | 물리적 설치 장소 |
   | 최대 용량 | 필수 | 최대 계량 가능 중량 (kg) |
   | 최소 눈금 | 필수 | 최소 측정 단위 (kg) |
   | 통신 포트 | 필수 | RS-232C 연결 포트 정보 |
   | 검정 유효일 | 필수 | 계량기 검정 유효기간 만료일 |
   | 사용 여부 | 필수 | 사용/미사용 선택 |

4. **저장** 버튼을 클릭합니다.

> **주의**: 저울의 검정 유효기간이 만료되면 해당 계량기로 수행한 계량은 법적 효력이 없을 수 있습니다. 검정 유효일을 반드시 관리하십시오.

#### 4.6.4 공통코드 관리

공통코드 관리(`/master/code`)에서는 시스템 전반에서 사용하는 분류 코드를 관리합니다.

1. 좌측 메뉴에서 **기준정보 > 공통코드**를 클릭합니다.
2. 코드 그룹 목록이 표시됩니다.
3. 코드 그룹을 선택하면 해당 그룹에 속한 상세 코드 목록이 표시됩니다.

주요 공통코드 그룹:

| 코드 그룹 | 설명 | 예시 |
|-----------|------|------|
| ITEM_TYPE | 품목 유형 | 부산물, 폐기물, 부재료, 반출, 일반 |
| WEIGH_MODE | 계량 모드 | LPR(자동), 모바일(OTP), 수동(manual) |
| DISPATCH_STATUS | 배차 상태 | 등록, 진행중, 완료, 취소 |
| VEHICLE_TYPE | 차량 유형 | 덤프트럭, 카고트럭, 탱크로리 등 |
| GATE_STATUS | 출문 상태 | 대기, 승인, 반려 |

> **주의**: 공통코드의 변경은 시스템 전체에 영향을 미칩니다. 변경 전 반드시 영향 범위를 확인하고, 필요 시 관리자와 협의하십시오.

### 4.7 통계

통계(`/statistics`) 기능은 계량 데이터를 다양한 기준으로 분석하여 보고서 형태로 제공합니다.

#### 4.7.1 일별 통계

1. 좌측 메뉴에서 **통계**를 클릭합니다.
2. 상단 탭에서 **일별**을 선택합니다.
3. 조회 기간(시작일 ~ 종료일)을 설정합니다.
4. **조회** 버튼을 클릭합니다.
5. 일자별 계량 건수, 총 중량, 품목별 비율이 표와 차트로 표시됩니다.

#### 4.7.2 월별 통계

1. 상단 탭에서 **월별**을 선택합니다.
2. 조회 연도와 월 범위를 설정합니다.
3. **조회** 버튼을 클릭합니다.
4. 월별 추이 차트와 함께 상세 데이터가 표시됩니다.

#### 4.7.3 품목별 통계

1. 상단 탭에서 **품목별**을 선택합니다.
2. 조회 기간과 품목 유형(복수 선택 가능)을 설정합니다.
3. **조회** 버튼을 클릭합니다.
4. 품목별 계량 건수, 중량 합계, 비율이 표시됩니다.

#### 4.7.4 업체별 통계

1. 상단 탭에서 **업체별**을 선택합니다.
2. 조회 기간과 업체(복수 선택 가능)를 설정합니다.
3. **조회** 버튼을 클릭합니다.
4. 업체별 계량 실적이 표시됩니다.

#### 4.7.5 Excel 다운로드

모든 통계 화면에서 조회 결과를 Excel 파일로 다운로드할 수 있습니다.

1. 원하는 통계를 조회합니다.
2. 화면 우측 상단의 **Excel 다운로드** 버튼을 클릭합니다.
3. 현재 조회 조건에 해당하는 데이터가 `.xlsx` 파일로 다운로드됩니다.
4. 파일명은 `계량통계_[유형]_[조회기간].xlsx` 형식입니다.

> **참고**: 대량의 데이터를 다운로드할 경우 시간이 소요될 수 있습니다. 브라우저의 다운로드 진행 상태를 확인하십시오.

### 4.8 계량대 실시간 현황

계량대 실시간 현황(`/weighing-station`) 화면은 현재 계량대에서 진행 중인 계량 작업을 실시간으로 모니터링하는 기능입니다.

#### 계량대 현황 확인

1. 좌측 메뉴에서 **계량 관리** 하위의 **계량대 현황**을 클릭합니다.
2. 각 계량대의 현재 상태가 카드 형태로 표시됩니다.

| 표시 항목 | 설명 |
|-----------|------|
| **계량대 명칭** | 계량대 이름 및 위치 |
| **현재 상태** | 대기 중, 계량 진행 중, 오류 등 |
| **차량 정보** | 현재 계량 중인 차량 번호 및 업체명 |
| **중량 값** | 실시간 중량 표시 (WebSocket 기반) |
| **계량 모드** | LPR(자동), OTP(모바일), 수동 |
| **진행 단계** | 1차/2차/3차 계량 구분 |

3. 계량이 완료되면 자동으로 상태가 갱신됩니다.

> **참고**: 이 화면은 WebSocket 기반으로 실시간 업데이트됩니다. 별도의 새로고침 없이 최신 상태가 반영됩니다.

### 4.9 모니터링

모니터링(`/monitoring`) 기능은 현장 장비의 실시간 상태를 확인하는 기능입니다.

#### 장비 상태 확인

1. 좌측 메뉴에서 **모니터링**을 클릭합니다.
2. 현장 장비 목록과 상태가 표시됩니다.

| 상태 | 표시 | 설명 |
|------|------|------|
| **온라인 (Online)** | 녹색 표시 | 장비가 정상 작동 중 |
| **오프라인 (Offline)** | 회색 표시 | 장비와 통신이 끊긴 상태 |
| **오류 (Error)** | 적색 표시 | 장비에 이상이 발생한 상태 |

모니터링 대상 장비:

| 장비 유형 | 설명 |
|-----------|------|
| **계량 지시기** | RS-232C 통신으로 중량 데이터를 읽는 장비 |
| **LPR 카메라** | 차량 번호판을 촬영하는 카메라 |
| **LiDAR 센서** | 차량 진입을 감지하는 센서 |
| **LED 전광판** | 운전자에게 안내 메시지를 표시하는 장비 |
| **차단기** | 계량대 입출구 차단기 |

3. 특정 장비를 클릭하면 상세 상태 정보를 확인할 수 있습니다.

   | 상세 정보 | 설명 |
   |-----------|------|
   | 장비명 | 장비 이름 및 코드 |
   | 설치 위치 | 물리적 위치 |
   | 현재 상태 | 온라인/오프라인/오류 |
   | 마지막 통신 | 최종 통신 성공 시각 |
   | 오류 내용 | 오류 상태인 경우 오류 내용 |

> **주의**: 장비 상태가 **오프라인** 또는 **오류**인 경우, 해당 계량대의 자동 계량이 정상적으로 수행되지 않을 수 있습니다. 즉시 관리자 또는 장비 담당자에게 연락하십시오.

> **참고**: 모니터링 화면은 WebSocket 기반으로 실시간 업데이트됩니다. 별도의 새로고침 없이 최신 상태가 반영됩니다.

## 5. 공통 기능

운전자와 담당자 모두 사용할 수 있는 공통 기능입니다.

### 5.1 마이페이지

마이페이지(`/mypage`)에서는 개인정보를 확인하고 비밀번호를 변경할 수 있습니다.

#### 5.1.1 개인정보 확인

1. 웹: 우측 상단 **사용자 아이콘** 클릭 후 **마이페이지** 선택
   모바일: 하단 내비게이션에서 **마이** 탭 선택
2. 개인정보가 표시됩니다.

   | 항목 | 설명 |
   |------|------|
   | 사용자 ID | 로그인 아이디 (변경 불가) |
   | 이름 | 사용자 이름 |
   | 역할 | 관리자, 담당자, 운전자 |
   | 소속 | 소속 업체 또는 부서 |
   | 연락처 | 등록된 전화번호 |
   | 이메일 | 등록된 이메일 주소 |

#### 5.1.2 알림 설정

마이페이지에서 알림 수신 여부를 설정할 수 있습니다.

1. 마이페이지 화면에서 **알림 설정** 항목을 확인합니다.
2. 다음 알림 유형별로 수신 여부를 설정할 수 있습니다.

   | 알림 유형 | 설명 |
   |-----------|------|
   | 배차 알림 | 새로운 배차 배정 시 알림 수신 |
   | 계량 완료 | 계량 완료 시 알림 수신 |
   | 계량표 발행 | 전자 계량표 생성 시 알림 수신 |
   | 공지사항 | 새 공지사항 등록 시 알림 수신 |
   | 시스템 알림 | 계정 관련, 시스템 점검 등 알림 수신 |

3. 원하는 알림의 토글 스위치를 켜거나 끕니다.
4. 변경 사항은 즉시 저장됩니다.

> **참고**: 모바일 앱의 푸시 알림을 받으려면 스마트폰 설정에서 앱 알림 권한이 허용되어 있어야 합니다. 앱 최초 실행 시 푸시 알림 토큰이 자동으로 등록됩니다.

#### 5.1.3 비밀번호 변경

1. 마이페이지 화면에서 **비밀번호 변경** 버튼을 클릭(탭)합니다.
2. 다음 정보를 입력합니다.

   | 입력 항목 | 설명 |
   |-----------|------|
   | 현재 비밀번호 | 현재 사용 중인 비밀번호 입력 |
   | 새 비밀번호 | 변경할 비밀번호 입력 |
   | 새 비밀번호 확인 | 새 비밀번호 재입력 |

3. **변경** 버튼을 클릭(탭)합니다.
4. "비밀번호가 변경되었습니다." 메시지가 표시되면 변경이 완료됩니다.

비밀번호 규칙:

| 규칙 | 설명 |
|------|------|
| 최소 길이 | 8자 이상 |
| 문자 조합 | 영문 대소문자, 숫자, 특수문자 중 3가지 이상 조합 |
| 이전 비밀번호 | 최근 3회 사용한 비밀번호는 재사용 불가 |
| 변경 주기 | 90일마다 변경 권장 (만료 7일 전 알림) |

> **주의**: 비밀번호를 변경하면 현재 세션이 종료됩니다. 새 비밀번호로 다시 로그인하십시오.

### 5.2 알림

알림 기능을 통해 시스템에서 발송하는 주요 안내 사항을 확인할 수 있습니다.

#### 알림 확인 방법

**웹**:
1. 상단 헤더의 **알림 아이콘**(종 모양)을 클릭합니다.
2. 읽지 않은 알림 수가 아이콘에 배지로 표시됩니다.
3. 알림 목록이 드롭다운으로 표시됩니다.
4. 알림을 클릭하면 해당 화면으로 이동합니다.

**모바일**:
1. 상단 바의 **알림 아이콘**을 탭합니다.

3. 알림을 탭하면 해당 상세 화면으로 이동합니다.

> **참고**: 모바일 앱의 경우, 앱이 백그라운드 상태에 있어도 푸시 알림을 통해 중요 안내를 받을 수 있습니다. 스마트폰 설정에서 앱 알림을 허용해 두십시오.

### 5.3 공지사항

공지사항(`/notice`)에서는 시스템 관련 안내, 점검 일정, 운영 변경사항 등을 확인할 수 있습니다.

4. 상단 **검색창**에 키워드를 입력하여 공지사항을 검색할 수 있습니다.
5. 공지사항을 클릭(탭)하면 상세 내용이 표시됩니다.
6. 첨부파일이 있는 경우 다운로드할 수 있습니다.

## 6. 자주 묻는 질문 (FAQ)

### 로그인/계정

**Q1. 비밀번호를 잊어버렸습니다. 어떻게 해야 하나요?**

A. 로그인 화면의 **비밀번호 찾기**를 클릭하여 등록된 이메일 또는 휴대전화로 임시 비밀번호를 발급받으십시오. 임시 비밀번호로 로그인 후 마이페이지에서 비밀번호를 반드시 변경하십시오. 자체적으로 해결이 어려운 경우 관리자에게 초기화를 요청하십시오.

**Q2. 계정이 잠겼다는 메시지가 표시됩니다.**

A. 비밀번호를 5회 연속 잘못 입력하면 계정이 30분간 잠금 처리됩니다. 30분 후에 올바른 비밀번호로 다시 시도하십시오. 긴급한 경우 관리자에게 즉시 잠금 해제를 요청하십시오.

**Q3. 모바일 앱에서 자동 로그인이 되지 않습니다.**

A. 자동 로그인은 마지막 로그인 후 7일간 유효합니다. 7일이 경과한 경우 다시 로그인이 필요합니다. 앱을 삭제 후 재설치한 경우에도 재로그인이 필요합니다.

### 배차

**Q4. 배차 목록에 내 배차가 보이지 않습니다.**

A. 다음 사항을 확인하십시오.
- 로그인한 계정이 해당 배차에 배정된 운전자의 계정이 맞는지 확인합니다.
- 배차 목록의 필터(상태, 날짜)가 올바르게 설정되어 있는지 확인합니다.
- 배차가 취소(CANCELLED)되었을 수 있으므로 취소 상태를 포함하여 조회합니다.
- 문제가 지속되면 계량 담당자에게 문의하십시오.

**Q5. 배차 상태가 변경되지 않습니다.**

A. 배차 상태는 계량 진행 상황에 따라 시스템이 자동으로 변경합니다. 1차 계량 시작 시 자동으로 "진행중"으로, 최종 계량 완료 시 "완료"로 변경됩니다. 수동으로 변경해야 하는 경우 담당자가 배차 상세 화면에서 상태를 변경할 수 있습니다.

### 계량

**Q6. 계량대에 진입했는데 자동 인식이 되지 않습니다.**

A. 다음 사항을 확인하십시오.
1. 차량 번호판이 깨끗하고 가려진 부분이 없는지 확인합니다.
2. 계량대 위에 차량이 정확하게 정차해 있는지 확인합니다.
3. LED 전광판의 안내 메시지를 확인합니다.
4. LPR 인식 실패 시 LED 전광판에 OTP 코드가 표시되면 모바일 OTP 계량으로 진행합니다.
5. OTP 계량도 불가한 경우 물류관제실에 연락하여 수동 계량을 요청합니다.

**Q7. OTP 코드를 입력했는데 "잘못된 코드"라고 나옵니다.**

A. 다음 사항을 확인하십시오.
- LED 전광판에 표시된 코드를 정확하게 입력했는지 확인합니다 (6자리).
- OTP 발급 후 5분이 경과하면 코드가 만료됩니다. 새로운 OTP를 확인하십시오.
- 3회 연속 오류 시 해당 OTP가 무효화됩니다. 물류관제실에 문의하십시오.

**Q8. 순중량이 맞지 않는 것 같습니다.**

A. 순중량은 |1차 중량 - 2차 중량|으로 자동 계산됩니다. 계량 상세 화면에서 각 차수별 중량을 확인하십시오. 중량에 이상이 있다고 판단되는 경우 담당자에게 재계량을 요청하십시오.

### 전자 계량표

**Q9. 전자 계량표가 생성되지 않습니다.**

A. 전자 계량표는 최종 계량(일반적으로 2차 계량)이 완료된 후 자동으로 생성됩니다. 계량이 아직 진행 중인 상태에서는 계량표가 발행되지 않습니다. 모든 계량이 완료되었음에도 계량표가 없는 경우 계량 담당자에게 문의하십시오.

**Q10. 전자 계량표를 카카오톡으로 공유했는데 상대방이 확인할 수 없다고 합니다.**

A. 공유된 링크는 인터넷 연결이 필요합니다. 수신자가 링크를 클릭할 때 인터넷에 연결되어 있는지 확인하십시오. 문제가 지속되면 SMS로 재공유를 시도하십시오.

### 출문

**Q11. 출문이 반려되었습니다. 어떻게 해야 하나요?**

A. 출문 반려 알림에 포함된 반려 사유를 확인하십시오. 계량 미완료, 서류 미비 등 사유에 따라 필요한 조치를 완료한 후 출문을 재요청하거나 계량 담당자에게 문의하십시오.

### 기타

**Q12. 모바일 앱에서 알림이 오지 않습니다.**

A. 스마트폰 설정에서 부산 스마트 계량 앱의 알림 권한이 허용되어 있는지 확인하십시오. Android의 경우 "설정 > 앱 > 부산 스마트 계량 > 알림"에서, iOS의 경우 "설정 > 알림 > 부산 스마트 계량"에서 확인할 수 있습니다.

## 7. 문제 해결 가이드

이 장에서는 시스템 사용 중 발생할 수 있는 주요 문제 상황과 해결 방법을 안내합니다.

### 7.1 로그인 관련 문제

| 증상 | 원인 | 해결 방법 |
|------|------|-----------|
| "아이디 또는 비밀번호가 일치하지 않습니다" | 아이디/비밀번호 오입력 | Caps Lock 확인 후 재입력. 비밀번호 찾기 기능 이용 |
| "계정이 잠겼습니다" | 5회 연속 비밀번호 오류 | 30분 대기 후 재시도 또는 관리자에게 해제 요청 |
| 로그인 후 바로 로그아웃됨 | 액세스 토큰 발급 오류 | 브라우저 쿠키/캐시 삭제 후 재시도 |
| "세션이 만료되었습니다" | 30분 이상 미사용 | 다시 로그인 (7일 이내 자동 갱신 실패 시) |
| 모바일 앱 로그인 실패 | 네트워크 연결 불량 | Wi-Fi 또는 모바일 데이터 연결 확인 후 재시도 |

### 7.2 계량 관련 문제

| 증상 | 원인 | 해결 방법 |
|------|------|-----------|
| LPR 인식 실패 | 번호판 오염/손상/가림 | 모바일 OTP 계량으로 전환. 번호판 청소 후 재시도 |
| OTP 입력 실패 | 코드 만료 또는 오입력 | 5분 이내 새 OTP 확인 후 재입력. 3회 실패 시 물류관제실 연락 |
| 중량이 0으로 표시됨 | 계량 지시기 통신 오류 | 물류관제실에 장비 점검 요청. 수동 계량으로 전환 |
| "배차를 찾을 수 없습니다" | 미등록 차량/배차 없음 | 담당자에게 배차 등록 여부 확인. 차량 번호 등록 확인 |
| 계량 데이터 미저장 | 서버 통신 오류 | 네트워크 연결 확인. 재시도. 지속 시 물류관제실 연락 |
| "다른 배차가 진행 중입니다" | 동일 차량 배차 중복 | 이전 배차 완료 또는 취소 후 재시도 |

### 7.3 전자 계량표 관련 문제

| 증상 | 원인 | 해결 방법 |
|------|------|-----------|
| 계량표가 생성되지 않음 | 계량 미완료 | 모든 차수의 계량이 완료되었는지 확인 |
| 계량표 내용 오류 | 입력 데이터 오류 | 담당자에게 수정 요청 |
| 카카오톡 공유 실패 | 카카오톡 미설치/권한 부족 | 카카오톡 앱 설치 확인. SMS 공유로 대체 |
| 공유 링크 접근 불가 | 네트워크 오류 | 인터넷 연결 확인. 시간 후 재시도 |

### 7.4 모바일 앱 관련 문제

| 증상 | 원인 | 해결 방법 |
|------|------|-----------|
| 앱이 실행되지 않음 | OS 버전 미충족 | Android 10+ 또는 iOS 14+ 확인. 앱 업데이트 |
| 앱 사용 중 멈춤 | 메모리 부족 | 앱 강제 종료 후 재실행. 기기 재시작 |
| 데이터 로딩 실패 | 네트워크 불안정 | Wi-Fi 또는 데이터 연결 확인. 안정적인 환경에서 재시도 |
| 알림 미수신 | 알림 권한 미허용 | 기기 설정에서 앱 알림 권한 허용 |
| 화면 표시 이상 | 앱 버전 구버전 | 앱 스토어에서 최신 버전으로 업데이트 |

### 7.5 웹 관련 문제

| 증상 | 원인 | 해결 방법 |
|------|------|-----------|
| 화면이 깨짐 | 미지원 브라우저 | Chrome 90 이상 사용 권장 |
| 실시간 데이터 미갱신 | WebSocket 연결 끊김 | 브라우저 새로고침(F5). 지속 시 관리자 문의 |
| Excel 다운로드 실패 | 팝업 차단 | 브라우저 팝업 차단 해제 확인 |
| 페이지 로딩 느림 | 대용량 데이터 조회 | 검색 조건(기간)을 좁혀서 재조회 |
| "권한이 없습니다" | 역할 권한 부족 | 해당 기능에 접근 권한이 있는지 관리자 확인 |

### 7.6 장비 모니터링 관련 문제

| 증상 | 원인 | 해결 방법 |
|------|------|-----------|
| 장비 상태 "오프라인" | 통신 장애 | 관리자 또는 장비 담당자에게 점검 요청 |
| 장비 상태 "오류" | 장비 이상 발생 | 오류 내용 확인 후 장비 담당자에게 수리 요청 |
| LiDAR 미감지 | 센서 오작동 | 물류관제실에 연락. 수동 계량으로 전환 |
| LED 전광판 표시 이상 | 통신 오류 | 물류관제실에 연락. 구두 안내 받아 진행 |
| 차단기 미작동 | 기계적 이상 | 현장 안전요원에게 즉시 연락 |

### 7.7 긴급 연락처

| 구분 | 연락처 | 담당 업무 |
|------|--------|-----------|
| **물류관제실** | 내선번호 확인 | 계량 운영, OTP 오류, 장비 이상 |
| **자재창고** | 내선번호 확인 | 품목 관련, 상/하차 관련 |
| **시스템 관리자** | 내선번호 확인 | 계정 관리, 시스템 오류 |
| **장비 담당자** | 내선번호 확인 | 장비 수리, 하드웨어 점검 |

> **참고**: 구체적인 연락처는 현장 게시판 또는 공지사항에서 확인하십시오.

## 8. 용어집

본 시스템에서 사용되는 주요 용어를 가나다순으로 정리합니다.

| 용어 | 영문 | 설명 |
|------|------|------|
| **게이트패스 (출문증)** | Gate Pass | 차량이 제철소를 퇴장할 때 필요한 허가 문서. 담당자 승인 필요 |
| **공차 중량** | Tare Weight | 화물을 적재하지 않은 빈 차량의 중량 |
| **계량 모드** | Weighing Mode | 계량 수행 방식. LPR(자동), 모바일(OTP), 수동(manual)의 세 가지 모드 |
| **계량 지시기 (인디케이터)** | Weighing Indicator | 계량대(로드셀)에서 측정된 중량을 표시하고 데이터를 전송하는 장비 |
| **기준정보** | Master Data | 시스템 운영의 기초가 되는 데이터. 업체, 차량, 저울, 공통코드 등 |
| **대시보드** | Dashboard | 주요 현황 정보를 한눈에 볼 수 있도록 구성한 종합 화면 |
| **라이다 (LiDAR)** | Light Detection and Ranging | 레이저를 이용하여 물체까지의 거리를 측정하는 센서. 차량 진입 감지에 사용 |
| **로드셀** | Load Cell | 중량을 전기 신호로 변환하는 센서. 계량대에 내장 |
| **반출** | Export | 제철소 내에서 외부로 자재를 반출하는 것 |
| **배차** | Dispatch | 차량에 운송 건을 배정하는 것 |
| **부산물** | Byproduct | 제철 과정에서 주산물 외에 부수적으로 생산되는 물질 (슬래그, 더스트 등) |
| **부재료** | Sub-material | 제철 과정에서 보조적으로 투입되는 원료 |
| **순중량** | Net Weight | 실제 화물의 중량. |1차 중량 - 2차 중량|으로 계산 |
| **실차 중량** | Gross Weight | 화물을 적재한 상태의 차량 전체 중량 |
| **액세스 토큰** | Access Token | 로그인 후 서버에 요청할 때 인증에 사용되는 임시 키. 유효기간 30분 |
| **웹소켓 (WebSocket)** | WebSocket | 서버와 클라이언트 간 양방향 실시간 통신을 지원하는 프로토콜 |
| **전자 계량표** | Electronic Weighing Slip (e-Slip) | 계량 결과를 전자 문서로 발행한 것. 종이 계량표를 대체 |
| **차수** | Measurement Order | 계량 순서. 1차(첫 번째 측정), 2차(두 번째 측정), 3차(세 번째 측정) |
| **폐기물** | Waste | 제철 과정에서 발생하는 폐기 대상 물질 |
| **LPR** | License Plate Recognition | 차량 번호판 자동 인식 기술. 카메라로 촬영한 이미지를 AI가 분석하여 번호판을 판독 |
| **OTP** | One-Time Password | 일회용 비밀번호. 6자리 숫자로 구성되며 5분간 유효 |
| **RS-232C** | RS-232C | 계량 지시기와 서버 간 직렬 통신 규격. 중량 데이터 전송에 사용 |
| **JWT** | JSON Web Token | 사용자 인증에 사용되는 토큰 규격. 액세스 토큰(30분)과 리프레시 토큰(7일) 구성 |
| **LED 전광판** | LED Display Board | 계량대 현장에서 운전자에게 안내 메시지를 표시하는 전자 표시 장치 |

| 항목 | 설명 |

> **엑셀 다운로드**: 검색 결과를 엑셀 파일로 다운로드할 수 있습니다.

### B.4 통계/보고서 페이지

**통계** 메뉴에서 다양한 차트와 보고서를 확인할 수 있습니다.
